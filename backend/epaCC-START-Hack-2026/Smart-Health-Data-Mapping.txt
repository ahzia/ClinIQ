The Problem: Data Everywhere, But Isolated
Every healthcare system generates enormous volumes of data — but those systems rarely speak to each other. Combining sources for meaningful analytics still requires many manual steps, making the process slow, error-prone, and difficult to scale.
Hospital Information Systems
Core clinical records in different formats.
Lab Software
Diagnostic results locked inside isolated databases, rarely accessible in real time.
Excel Exports
Manual data extracts with inconsistent structure, prone to versioning and mapping errors.
Sensor & Free Text
Unstructured data from devices and clinical notes that resists automated processing entirely.
5

Real-World Healthcare Data
Across care facilities, data is generated continuously in six core categories — each critical for delivering quality care, yet each stored independently across disconnected systems. Understanding these data types is the first step to unifying them.
Care Assessments
Structured nursing evaluations capturing patient condition, needs, and care plan progress over time.
Diagnoses
ICD-coded or free-text clinical diagnoses recorded at admission, discharge, and throughout the care pathway.
Lab Parameters
Quantitative results from blood work, urinalysis, and other diagnostic tests — often in isolated lab systems.
Vital Signs
Continuous or episodic measurements including pulse, blood pressure, temperature, and oxygen saturation.
Medication Plans
Inpatient and outpatient drug regimens with dosage, frequency, and administration records.
Resource Planning
Staff assignments, bed management, and operational scheduling data that intersects with clinical workflows.

Header Schema: The Raw Data Reality
The challenge begins with the data itself. Below are three CSV export formats from the System — each representing the same assessments structured differently. Participants will work with all three formats and must bridge these structural gaps programmatically.
epaAC-Data-1.csv
One record per assessment item (Pro Item 1 Dataset). Each row represents a single scored item with its station, account, and SID value.
epaAC-Data-2.csv
One record per full assessment with all item data as columns — SID-based wide format. Extremely wide rows, one row per evaluation episode.
epaAC-Data-3.csv
One record per assessment with text-based headers and IID-based columns — the most human-readable but least machine-friendly format.
7

Database Design
The underlying relational schema organizes incoming data from all source systems into a unified case-centric structure. The central tbCaseData table anchors all imports — connecting assessments, lab results, nursing reports, medication records, and motion sensor data into a single patient timeline.
Each import table (tbImportAcData, tbImportLabsData, tbImportNursingDailyReports, tbImportMedicationInpatientData, tbImportDeviceMotionData) links to the core case record via shared keys — forming the data model your mapping solution must target.

In Addition: Interactive Dashboard
Beyond data mapping, your solution should surface insights through a purpose-built dashboard that gives clinical and operational teams full visibility into the quality, provenance, and integrity of their data — in real time.
Data Origin & Structure
Visualize where each data field comes from, which source system it originated in, and how it was mapped to the unified model.
Quality & Completeness
Surface gaps, missing values, inconsistencies, and field-level coverage metrics across all imported datasets.
Anomaly Detection
Automatically flag unusual patterns, outliers, or values that fall outside expected clinical or statistical ranges.
Alerts & Corrections
Proactive alerts for mapping issues combined with an intuitive interface for reviewing and correcting flagged records.

Technology: Your Choice
Freedom to Innovate
Use any technology stack, framework, or approach that works best for your team — AI/ML models, NLP pipelines, LLM-based agents, rule-based logic, or any combination thereof. There are no restrictions on language or tooling.
One Key Requirement
Your solution must be deployable on-premises. Healthcare data is sensitive — it must remain securely within the institution's own infrastructure at all times. No cloud-only solutions will be accepted.
Whether you're leveraging transformer models for entity extraction, building deterministic schema-matching pipelines, or designing a hybrid agent architecture — the best approach is the one that solves the problem reliably, securely, and at scale.
